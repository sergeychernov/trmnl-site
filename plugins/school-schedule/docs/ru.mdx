# Школьное расписание

Плагин **«Школьное расписание»** показывает на TRMNL полное расписание уроков на неделю: дни, номера уроков, названия предметов и время начала/окончания.  
Он подходит для школьников, родителей и учителей — как «умный» заменитель бумажного расписания рядом с рабочим столом или у двери.

## Что делает плагин

- **Показывает недельное расписание**  
  Дни недели (понедельник–суббота) с уроками по порядку.
- **Отображает время уроков**  
  По единой сетке начала уроков (`lessonStarts`) и длительности урока (`lessonDurationMin`).
- **Гибко заполняется под конкретный класс**  
  Для каждого дня можно задать свой список предметов: от пустого дня до полной загрузки.

## Основные настройки

Тип настроек: `ScheduleSettings`.

- **Часовой пояс (`timeZone`)**
  - Формат IANA, например: `Europe/Moscow`.  
  - Используется для вычисления текущего времени и корректного отображения времён уроков.

- **Длительность урока (`lessonDurationMin`)**
  - Целое число минут, например `45`.  
  - По нему считается время окончания каждого урока.

- **Начала уроков (`lessonStarts`)**
  - Массив строк в формате `HH:MM`, общий для всех дней недели.  
  - Примеры: `["08:30", "09:25", "10:30", ...]`.  
  - Количество элементов определяет максимальное число уроков в дне.

- **Показывать время (`showTimes`)**
  - `true` — в расписании видно время начала/конца каждого урока.  
  - `false` — только названия предметов и номера уроков.

- **Расписание по дням (`schedule`)**
  - `mon`, `tue`, `wed`, `thu`, `fri`, `sat?` — массивы строк с названиями предметов.  
  - Примеры:
    - Понедельник: `["Математика", "Русский язык", "История", "Английский"]`
    - Вторник: `["Литература", "Физкультура", "Физика", "Труд"]`
  - Пустой массив для дня означает «уроков нет».

## Типичные сценарии использования

- Экран рядом с рабочим местом школьника: «какие уроки сегодня и во сколько».
- Расписание младших детей на TRMNL в семье, чтобы родители и дети видели план дня.

## Ограничения и рекомендации

- Плагин **не обращается к внешним API** — все данные задаются вручную в настройках.
- Важно, чтобы:
  - все значения `lessonStarts` были в формате `HH:MM`,
  - во всех массивах `schedule.*` были только строки (названия предметов).
- Количество предметов в дне не должно превышать длину массива `lessonStarts`, иначе «лишние» уроки не будут корректно привязаны ко времени.
