## Telegram plugin для TRMNL

Плагин `telegram` показывает на экране TRMNL сообщения, которые Telegram‑бот TRMNL отправляет на это устройство.

### Как работает плагин

- Плагин рендерится как обычный React‑компонент (`Render.tsx`), который затем конвертируется в 1‑битное BMP‑изображение OG‑пайплайном.
- В `context.telegramId` передаётся Telegram ID владельца устройства:
  - ID берётся из коллекций `device_members` и `accounts` (связка `userId` ↔ `provider: "telegram"`).
  - Если Telegram аккаунт **не привязан**, плагин выводит сообщение:
    > «Для работы этого плагина привяжите Telegram аккаунт в профиле пользователя».
- Если Telegram привязан, плагин показывает заглушку:
  - заголовок **Telegram TRMNL**;
  - текст:
    > «Здесь будут выводиться сообщения, которые бот TRMNL отправляет на это устройство».

Фактический вывод истории сообщений будет добавлен позже, когда появится серверное хранилище событий от бота.

### Настройки

- Тип настроек: `TelegramSettings = Record<string, never>` — у плагина **нет настраиваемых параметров**.
- `Editor.tsx` показывает короткий текст «Этот плагин пока не имеет настроек».

### Когда используется

- При рендере устройства через:
  - `/api/render/device` (основной BYOS‑рендер),
  - `/api/render/plugin?plugin=telegram&...` (предпросмотр плагина).
- Плагин регистрируется в `plugins/index.ts` и может быть выбран на странице настроек устройства.

### Поведение при ошибках

- В случае проблем OG‑рендера в логах сервера будут записи:
  - `[telegram-plugin] render { width, height, hasUser, hasTelegramId }`;
  - `[render-plugin] error { pluginId: "telegram", ... }` из `/api/render/plugin`.
